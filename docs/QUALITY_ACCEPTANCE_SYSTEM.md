# 音乐质量、受众与授权验收体系

## 1. 总验收规则

评分采用 `0-5` 分：

- `0`：缺失。
- `1`：严重不合格。
- `2`：勉强可听或可用。
- `3`：达到交付底线。
- `4`：商业可用。
- `5`：同类优秀作品水平。

通过规则：

| 项目 | 规则 |
|---|---|
| 总分 | `>=80/100` 可验收；`>=90/100` 优秀；`70-79` 返工复验；`<70` 不通过 |
| 单项底线 | 任一一级项 `<3/5`，即使总分够也必须返工 |
| 一票否决 | 未授权采样、未授权人声克隆、明显抄袭、音频破损、严重削波、违法违规内容、授权文件缺失 |
| 证据留存 | 音频、歌词、MIDI/旋律线、分轨、母带检测、相似性检查、授权包、QA 报告 |

## 2. 100 分评分矩阵

| 一级项 | 权重 | 可执行判断标准 | 通过线 |
|---|---:|---|---|
| 好的音乐整体判断 | 15 | 情绪目标明确；结构完整；有记忆点；从头到尾无明显乏味段；听感符合 brief；有二次播放价值 | `>=12` |
| 朗朗上口 | 10 | 主 hook 首听可识别；副歌/主旋律可哼唱；重复与变化平衡；标题/关键词能被记住 | `>=8` |
| 旋律 | 10 | 旋律动机清楚；音域适合目标人群；节奏与歌词重音贴合；和弦内外音处理合理；有起承转合 | `>=8` |
| 歌词 | 10 | 主题清楚；语言自然；押韵不生硬；副歌有核心句；无陈词堆砌；无版权、敏感、低俗风险 | `>=8` |
| 编曲 | 10 | 段落推进合理；乐器分工清楚；能量变化明确；风格元素准确；不过度堆叠 | `>=8` |
| 混音母带 | 10 | 人声/主奏清晰；低频稳定；无刺耳、浑浊、削波；多设备可听；响度与用途匹配 | `>=8` |
| 风格/受众匹配 | 10 | 风格标签准确；目标用户反馈达标；使用场景匹配；不是泛音乐模板感 | `>=8` |
| 原创性与安全 | 5 | 无明显旋律、歌词、采样撞车；无未披露参考复刻；无违规内容 | `>=4` |
| 交付与授权 | 10 | 权利链完整；文件齐全；授权范围明确；元数据、署名、分成清楚 | `>=9` 且不可缺文件 |
| 网站 MVP | 10 | 每日任务、作品库、评分中心、交付授权中心可打开；核心流程可见；状态清楚 | `>=8` |

## 3. 好音乐判断标准

| 检查项 | 3 分底线 | 5 分优秀 |
|---|---|---|
| 情绪表达 | 能听出目标情绪 | 情绪递进自然，有高潮和释放 |
| 结构 | 段落完整，无断裂 | 段落推进让人愿意听完 |
| 记忆点 | 至少一个可辨识 hook | hook、节奏、音色或歌词有独特识别度 |
| 完成度 | 无明显错拍、跑调、空洞段 | 可直接进入商用剪辑或交付 |
| 重听价值 | 不反感二听 | 目标用户主动收藏、复听或愿意使用 |

## 4. 朗朗上口验收

| 检查项 | 验收方法 | 通过线 |
|---|---|---|
| 首听 hook 识别 | 目标用户听 1 遍后说出最记得的旋律或歌词 | `>=60%` 能指出同一 hook |
| 跟唱性 | 播放第 2 遍副歌时观察是否能跟唱或哼唱 | `>=50%` 可跟上主要轮廓 |
| 24 小时记忆 | 次日让用户回忆标题句或哼主旋律 | `>=30%` 能回忆核心片段 |
| 入口速度 | 主 hook 或强记忆点出现时间 | 流行/短视频 `<=30s`；广告 `<=5s` |
| 重复与变化 | hook 重复但不机械 | 重复 2-4 次，有音高、节奏、编曲或歌词变化 |

自动化辅助指标：

- hook 出现时间。
- 动机重复次数。
- 主旋律音域。
- 旋律轮廓重复度。
- 歌词核心句重复位置。
- 主副歌能量差。

## 5. 旋律验收

| 检查项 | 验收标准 |
|---|---|
| 音域 | 大众演唱主旋律建议控制在 `9-12` 个半音内；儿童歌更窄；专业演唱可放宽 |
| 动机 | 2-4 小节内出现清楚动机，并在副歌形成强化 |
| 轮廓 | 有上行、下行、停顿和解决，不是随机音符游走 |
| 和声关系 | 强拍多落在和弦音或稳定音；非和弦音有经过、倚音、解决逻辑 |
| 节奏 | 旋律节奏与 groove 一致，不抢歌词重音 |
| 可唱性 | 连续大跳有准备或解决；换气点合理；长音位置自然 |

返工触发：

- 音域过宽。
- 连续大跳过密。
- 旋律和歌词重音冲突。
- 主歌、副歌没有明显差异。
- hook 无法复述。

## 6. 歌词验收

| 检查项 | 验收标准 |
|---|---|
| 主题 | 一句话能说清这首歌在讲什么 |
| 副歌 | 有标题句、核心态度或可传播金句 |
| 语感 | 朗读不拗口，重音与旋律重拍一致 |
| 押韵 | 服务表达，不为押韵牺牲语义 |
| 画面 | 至少有 2-3 处具体意象，避免全是抽象词 |
| 原创性 | 不仿写热门歌词句式，不出现高相似连续句 |
| 安全 | 无侵权指名、仇恨、色情低俗、违法引导、未成年人不适内容 |

纯音乐任务不使用歌词评分；权重重新分配到旋律、结构、编曲和受众匹配。

## 7. 编曲验收

| 检查项 | 验收标准 |
|---|---|
| 结构图 | 明确标注 intro、verse、pre、chorus、bridge、outro 或对应结构 |
| 能量曲线 | 主歌留空间，副歌/Drop/高潮有明显提升 |
| 乐器分工 | 低频、节奏、中频和主旋律不互相抢占 |
| 过门转场 | 段落之间有 fill、riser、停顿、鼓花或和声转接 |
| 风格语言 | 使用该风格可信的鼓组、贝斯、和声、音色和速度区间 |
| 商用适配 | 能剪 15s、30s、60s 版本；广告/短视频要有前 3 秒抓点 |

## 8. 混音母带验收

| 检查项 | 验收标准 |
|---|---|
| 文件规格 | WAV/AIFF 优先；`44.1kHz` 或更高；母带建议 `24-bit` |
| 峰值 | Streaming 安全版 True Peak `<= -1 dBTP`；高响度版建议 `<= -2 dBTP` |
| 响度 | 默认流媒体安全版约 `-14 LUFS`；商业流行/EDM 可另交高响度版，但不得明显牺牲音质 |
| 动态 | 不因限制器导致泵动、失真、军鼓/人声塌陷 |
| 人声/主奏 | 手机、耳机、车载、小音箱均清楚 |
| 频段 | 低频不轰，中频不堵，高频不刺，齿音可控 |
| 兼容 | 单声道检查无核心元素消失 |
| 噪声 | 无爆音、断点、底噪突增、尾音截断 |

## 9. 类型与受众验收标准

| 类型/用途 | 目标受众 | 重点验收 | 受众测试 |
|---|---|---|---|
| 流行 Pop | 泛音乐听众 | 副歌 hook、标题句、现代音色、主唱清晰 | 首听后 `>=60%` 记住副歌 |
| 短视频/广告 | 剪辑、广告、短内容创作者 | 前 3-5 秒抓耳、口号适配、可循环剪辑 | 15 秒版本完整传达情绪和记忆点 |
| Hip-hop/Trap | 说唱、运动、街头内容 | flow、鼓和 808、押韵密度、态度一致 | 目标用户认为 groove 不松、不假 |
| EDM/舞曲 | 健身、派对、节奏剪辑 | build/drop、低频冲击、舞池能量 | drop 前后能量提升明显 |
| R&B/Soul | 年轻听众、情绪歌单 | 和声质感、律动、转音克制、空间感 | 听感顺滑，不机械量化过重 |
| 摇滚/金属 | 乐队、现场感用户 | riff、鼓贝斯冲击、吉他层次 | 高能段不糊，主 riff 可识别 |
| 民谣/唱作 | 叙事、情绪用户 | 词曲真诚、旋律自然、编曲不过度 | 用户能复述故事或情绪核心 |
| 国风/戏腔/民乐融合 | 中文内容、剧情剪辑 | 调式、器乐语汇、唱腔准确，避免廉价堆砌 | 目标用户不觉得伪国风模板 |
| 儿童/亲子 | 儿童和家长 | 音域窄、词简单、安全积极、节奏清楚 | 儿童能跟唱，家长不反感内容 |
| 游戏/BGM | 游戏、直播、互动内容 | 循环无缝、情绪稳定、可做分层 stem | 循环 10 分钟不烦，不抢交互声音 |
| 影视/预告 | 短剧、广告、宣传片 | 情绪叙事、动态跨度、hit point | 能支撑剪辑节奏，不喧宾夺主 |
| Lo-fi/Ambient | 学习、办公、放松 | 质感、空间、低干扰、长听舒适 | 背景播放不刺耳、不突然跳出 |

## 10. 原创性与安全验收

一票否决：

- 未授权采样。
- 未授权真实人声克隆。
- 明显复刻真实歌曲旋律。
- 明显复写真实歌词。
- 以真实艺人名义或声线误导。
- 参考音频高相似输出。
- 违法、仇恨、色情低俗、未成年人不适内容。

证据要求：

- 旋律相似度报告。
- 歌词相似度报告。
- 参考画像说明。
- 人声来源说明。
- sample/loop/preset 来源。
- 生成模型和 prompt 摘要。

## 11. 交付授权验收

| 项目 | 必须通过 |
|---|---|
| sample / loop | 每个 sample、loop、preset demo、MIDI 包均有可商用证明或来源记录 |
| 人声/声音克隆 | 不得使用未授权真人声音、艺人声线复刻、名人风格误导 |
| 训练/生成来源披露 | 记录模型、工具、提示词摘要、人工修改说明 |
| 人类贡献 | 明确人类在作词、作曲、编曲、选择、编辑、混音中的贡献 |
| 授权范围 | 明确地域、期限、媒介、用途、是否独占、是否可转授权、是否可商用、是否可改编 |
| 权利拆分 | 词、曲、编曲、录音制品/master、表演、制作人、平台服务权利分清 |
| 元数据 | 歌名、作者、制作人、歌词语言、显式内容标识、BPM、key、版本号 |
| 交付文件 | Master、伴奏、干声、分轨/stems、歌词、工程或 MIDI、封面、授权书、验收单 |

交付包清单：

- `Master WAV`：最终母带。
- `Instrumental`：无人声伴奏。
- `Vocal acapella`：如合同约定。
- `Stems`：drums、bass、music、lead、backing vocal、FX 等。
- `Lyrics`：最终歌词和版本记录。
- `Cue sheet / metadata`：时长、BPM、key、作者、版权方、用途。
- `License pack`：授权书、素材来源、第三方素材许可、声音授权、分成表。
- `Acceptance report`：评分表、母带检测、目标用户测试、缺陷与返工记录。

## 12. 网站 MVP 验收

第一阶段网站验收仍然聚焦音乐制作，不做音乐分发平台。

| 模块 | MVP 通过标准 |
|---|---|
| 每日任务 | 能看到 10-20 条自动制作任务、阶段、下一个 Agent、返工次数、QA 状态 |
| 作品库 | 能看到作品、版本、风格、BPM、key、最高分、资产状态、授权状态 |
| 评分中心 | 能看到待评分作品、分项评分、硬门槛、失败原因、返工建议 |
| 交付与授权中心 | 能看到交付包、授权主体、使用范围、AI 声明、导出规格、阻断原因 |
| LOOP 可视化 | 能看到状态流转、失败原因、责任 Agent、返工上限 |
| 页面边界 | 不出现社区、推荐流、歌单平台、点赞评论、粉丝关系 |

## 13. 验收检查清单

音乐创作检查：

- [ ] brief、目标风格、目标用户、用途、时长已确认。
- [ ] 歌词、旋律、编曲、混音分别评分。
- [ ] 主 hook、标题句、高潮段已标注。
- [ ] 相似性/抄袭风险已复核。
- [ ] 返工意见有责任 Agent 和复验条件。

音频技术检查：

- [ ] 无爆音、断点、削波、尾音截断。
- [ ] True Peak、LUFS、动态、频谱检查完成。
- [ ] 手机、耳机、车载/蓝牙音箱、小音箱至少 4 类设备试听。
- [ ] 单声道兼容检查通过。
- [ ] Master、伴奏、stems 命名统一且版本一致。

授权交付检查：

- [ ] 所有第三方素材来源可追溯。
- [ ] 未使用未授权人声、艺人声线、受保护录音片段。
- [ ] 授权书包含用途、期限、地域、独占性、转授权、改编权、署名、费用。
- [ ] 词曲、录音、表演、制作人权利边界清楚。
- [ ] 元数据、歌词、母带、分轨、授权包齐全。

网站 MVP 检查：

- [ ] 网站能打开。
- [ ] 能看到每日任务。
- [ ] 能看到作品库。
- [ ] 能看到评分中心。
- [ ] 能看到交付与授权中心。
- [ ] 每个页面表达音乐制作流程。
- [ ] 未做成音乐分发平台。
