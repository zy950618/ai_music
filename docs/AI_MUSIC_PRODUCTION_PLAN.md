## 1. 定位

本项目是 AI 音乐制作系统，不是音乐分发平台。

核心目标是每天自动完成一批音乐制作任务：策划、写词、作曲、编曲、生成候选、质量验收、返工、混音母带、入库、交付包生成。系统可以预留平台连接和授权配置，但不做社区、推荐流、歌单平台、用户评论、点赞、粉丝关系等音乐平台能力。

第一阶段验收口径：

- 网站能打开。
- 能看到 `每日任务`。
- 能看到 `作品库`。
- 能看到 `评分中心`。
- 能看到 `交付与授权中心`。
- `交付与授权中心`只管理导出包、授权配置、文件规格、元数据和可选平台连接，不表达成音乐分发平台。

## 2. 关键假设

- 每天自动创建 10-20 条音乐制作任务。
- 每条任务生成 3-5 个候选版本。
- 每条任务最多自动返工 3 轮。
- 普通制作流程不需要人工逐首点击。
- 人工只处理授权、账号连接、高风险版权判断、连续失败、审美冲突和系统异常。
- 参考歌曲只能用于结构、情绪、配器、BPM、能量曲线等风格画像分析，不能复制旋律、歌词、歌手声线或可识别作品特征。

## 3. 系统能力分层

### 3.1 制作层

- 自动生成每日制作任务。
- 自动生成制作 brief。
- 自动写词和改词。
- 自动规划旋律、和声、节奏、段落。
- 自动生成多版本候选音频。
- 自动混音、母带和导出。

### 3.2 验收层

- 音频质量验收。
- 旋律和朗朗上口验收。
- 歌词验收。
- 编曲验收。
- 风格匹配验收。
- 受众匹配验收。
- 原创性和版权风险验收。
- 制作交付完整性验收。

### 3.3 LOOP 层

- 低分作品自动分析失败原因。
- 根据失败原因生成返工指令。
- 按原因返工，不盲目重生。
- 达到返工上限后废弃或进入人工复核。

### 3.4 资产层

- 保存作品、版本、音频、分轨、歌词、MIDI、prompt、seed、模型版本、QA 报告、返工记录、授权信息。
- 支持按风格、受众、用途、状态、评分检索。

### 3.5 交付与授权层

- 生成交付包。
- 管理授权配置。
- 管理导出规格。
- 管理元数据。
- 管理可选平台连接配置。
- 不做音乐平台。

## 4. 多 Agent 与 Skills

| Agent / Skill | 职责 | 输入 | 输出 |
|---|---|---|---|
| `Production Planner Agent` | 每天决定做哪些音乐制作任务 | 库存缺口、风格配额、历史质量、制作目标 | 10-20 条制作任务 |
| `Brief Agent` | 把任务变成可执行制作说明 | 制作任务 | 结构、风格、BPM、调式、配器、验收阈值 |
| `Audience Profiler Skill` | 定义目标受众和使用场景 | 风格、用途、目标平台或客户场景 | 受众画像和喜好标准 |
| `Lyric Agent` | 写词、改词、押韵、hook 句 | 主题、语言、情绪、受众 | 歌词版本和可唱性说明 |
| `Composition Agent` | 规划旋律、和声、节奏骨架 | brief、歌词、风格 | 旋律方案、和弦、段落结构 |
| `Arrangement Agent` | 规划编曲和乐器层次 | 作曲方案、风格 | intro、verse、hook、bridge、outro 编曲说明 |
| `Sound Design Skill` | 音色、鼓组、合成器、环境声 | 编曲说明 | 音色方案 |
| `Generation Router` | 调度真实生成模型 | prompt、参数、模型配置 | 多个候选音频版本 |
| `Audio QA Skill` | 客观音频质量检查 | 候选音频 | 响度、峰值、静音、爆音、时长、结构检查 |
| `Music Quality Judge` | 主观音乐性评分规则化 | 音频、歌词、brief | 旋律、hook、编曲、完成度评分 |
| `Originality Guard` | 原创性与版权风险检查 | 音频、歌词、参考画像 | 相似度、模仿风险、风险等级 |
| `Rework Orchestrator` | 把低分原因转成返工动作 | QA 报告 | 返工 brief |
| `Mix Master Agent` | 混音和母带 | 选中版本、分轨 | 母带、预览版、分轨 |
| `Catalog Agent` | 入库与标签管理 | 已验收作品 | 作品资产记录 |
| `Delivery Auth Agent` | 交付包和授权配置 | 入库作品、授权模板 | 导出包、授权记录、元数据 |
| `Ops Reporter Agent` | 生成日报和异常清单 | 全流程状态 | 产能、通过率、失败原因、明日建议 |

## 5. 每日自动化 LOOP

```text
00:00 Production Planner 创建 10-20 条制作任务
00:10 Brief Agent 生成制作 brief
00:30 Lyric / Composition / Arrangement 生成制作方案
01:00 Generation Router 生成 3-5 个候选版本
03:00 Audio QA 做第一轮质量检查
04:00 Rework Orchestrator 对低分版本自动返工
06:00 第二轮生成和 QA
07:00 Originality Guard 做原创性和版权风险检查
08:00 Music Quality Judge 和 Audience Profiler 综合选优
09:00 Mix Master Agent 做混音母带
10:00 Final QA 做最终验收
10:30 Catalog Agent 入库
11:00 Delivery Auth Agent 生成交付包和授权记录
18:00 Ops Reporter 输出日报和次日任务权重建议
```

## 6. 状态流转

主路径：

```text
CREATED
  -> BRIEF_READY
  -> COMPOSITION_READY
  -> GENERATING
  -> VERSIONS_READY
  -> AUDIO_QA_PASS
  -> ORIGINALITY_PASS
  -> SELECTED
  -> MIXING
  -> MASTERED
  -> FINAL_QA_PASS
  -> STORED
  -> DELIVERY_PACKAGE_READY
```

返工路径：

```text
VERSIONS_READY
  -> AUDIO_QA_FAIL
  -> REWORK_PENDING
  -> REWORKING
  -> VERSIONS_READY
```

版权风险路径：

```text
AUDIO_QA_PASS
  -> ORIGINALITY_REWORK
  -> REWORK_PENDING
  -> REWORKING
  -> VERSIONS_READY
```

人工复核路径：

```text
ORIGINALITY_HIGH_RISK
  -> HUMAN_REVIEW_REQUIRED
  -> HUMAN_APPROVED / REJECTED / REBRIEF
```

失败终止路径：

```text
REWORKING
  -> REWORK_LIMIT_REACHED
  -> REJECTED
```

## 7. 返工规则

返工必须基于明确失败原因。

| 失败原因 | 自动返工动作 |
|---|---|
| 爆音、削波、响度异常 | 降低峰值、重做混音、重新母带 |
| 静音、断裂、生成瑕疵 | 换 seed 或重新生成该段 |
| hook 弱 | 重写副歌旋律、提前高潮、增强重复动机 |
| 旋律不顺口 | 降低音程跳跃、简化节奏、缩小音域 |
| 歌词不可唱 | 减少字数、重排重音、改押韵 |
| 风格跑偏 | 回到 brief 重新约束 BPM、乐器、情绪 |
| 编曲空洞 | 增加低频、节奏层、过门和段落推进 |
| 编曲太满 | 减少层数、突出主旋律和人声 |
| loop 不自然 | 重做尾部衔接和循环点 |
| 相似度风险 | 重写旋律轮廓、节奏型、歌词核心句 |
| 人声问题 | 重生成人声、换音域、改为纯音乐版本 |

限制：

- 单个版本最多自动返工 2 次。
- 单条任务最多自动返工 3 轮。
- 版权高风险不自动硬过，必须进入人工复核或废弃。

## 8. 质量验收标准

验收分两层：硬性门槛和加权评分。

### 8.1 硬性门槛

任何一项失败都不能进入交付包：

- 明显爆音、削波、大片静音、音频断裂。
- 歌词存在明显侵权、敏感或不可授权内容。
- 明确模仿特定真实歌手、真实歌曲或可识别旋律。
- 参考歌曲相似度高风险。
- 文件缺失或导出包不完整。
- 授权配置缺失时，不允许进入自动交付队列。

### 8.2 加权评分

| 维度 | 权重 | 判断重点 |
|---|---:|---|
| 听感与音频质量 | 15 | 无刺耳、无糊、无断裂、动态合理 |
| 旋律质量 | 20 | 有动机、有发展、可哼唱、音域合理 |
| 朗朗上口 | 15 | hook 早出现、重复有效、核心句好记 |
| 结构完整度 | 10 | intro、主段、副歌、过渡、结尾清楚 |
| 编曲与音色 | 10 | 层次清楚、主次明确、音色匹配风格 |
| 歌词与可唱性 | 10 | 押韵、重音、语义、可唱性、记忆点 |
| 受众匹配 | 10 | 符合目标人群场景和情绪需求 |
| 原创与安全 | 5 | 不撞歌、不冒名、不高风险模仿 |
| 制作交付完整性 | 5 | 元数据、分轨、母带、版本记录完整 |

评分结果：

- `90-100`：优先入库，可作为高质量样本。
- `80-89`：通过，进入交付包准备。
- `70-79`：自动返工一次。
- `<70`：重开任务或废弃。
- 硬性门槛失败：无论总分多少，不能通过。

## 9. 朗朗上口判断

朗朗上口不能只靠感觉，系统先用可计算规则做初筛：

| 指标 | 合格倾向 |
|---|---|
| hook 出现时间 | 前 15 秒内出现主旋律或副歌动机 |
| 动机长度 | 2-4 小节，便于记忆 |
| 动机重复 | 至少重复 2 次，并有轻微变化 |
| 音域 | 普通人可唱，避免频繁极端高低音 |
| 音程 | 大跳不过密，级进和小跳占主导 |
| 节奏 | 与歌词重音一致，不别扭 |
| 核心句 | 歌名或核心句在副歌中可识别 |
| 情绪峰值 | 有明确高潮，不全程平铺 |
| 循环性 | 短视频或 BGM 任务结尾能自然循环 |

## 10. 不同类型音乐的受众验收标准

| 类型 | 目标受众 | 验收重点 |
|---|---|---|
| 短视频 BGM | 剪辑、广告、短内容创作者 | 前 3 秒抓耳、15-30 秒高潮、节奏明确、可剪辑 |
| 流行人声 | 泛音乐听众 | 副歌记忆点、歌词共情、人声清晰、结构商业化 |
| R&B / Soul | 年轻听众、情绪歌单 | groove、和声色彩、人声细节、氛围感 |
| Hip-Hop / Trap | 说唱、运动、街头内容 | 鼓组冲击、低频、flow 空间、hook 口号 |
| Lo-fi 学习 | 学生、办公人群 | 不抢注意力、循环自然、低刺激、稳定律动 |
| 睡眠冥想 | 放松和睡眠场景 | 低动态、无突发、长时长、柔和频段 |
| 儿童歌曲 | 儿童和家长 | 旋律简单、歌词安全、重复强、正向主题 |
| 古典 / 钢琴 | 音乐学习、审美用户 | 主题发展、和声逻辑、演奏感、结构完整 |
| 国风 / 古风 | 中文内容、剧情剪辑 | 五声音阶倾向、民乐音色、诗性歌词、情绪克制 |
| 游戏音乐 | 游戏、直播、互动内容 | 可循环、层次清晰、场景标签明确、不过度抢戏 |
| 影视配乐 | 短剧、广告、宣传片 | 情绪递进、铺垫和爆发、画面适配 |
| 电子 / 运动 | 健身、派对、节奏剪辑 | BPM 稳、drop 清晰、低频有力、能量持续 |
| 企业 / 商业 | 品牌片、产品片 | 稳定、干净、积极、不过度个性化 |

## 11. 授权配置与交付中心边界

`交付与授权中心`不是音乐平台。它只解决制作完成后的资产交付问题。

包含：

- 作品授权状态。
- 授权模板。
- 使用范围。
- 授权主体。
- 模型和素材来源记录。
- AI 生成声明。
- 导出规格。
- 交付包内容。
- 可选平台连接配置。
- 交付队列和导出状态。

不包含：

- 音乐社区。
- 用户主页。
- 粉丝关系。
- 评论点赞。
- 推荐流。
- 在线音乐播放器平台。
- 榜单。
- 面向公众的分发市场。

授权配置建议字段：

| 字段 | 含义 |
|---|---|
| `rights_owner` | 权利归属主体 |
| `usage_scope` | 可使用范围，如内部、商用、客户交付、广告素材 |
| `territory` | 地区范围 |
| `duration` | 授权期限 |
| `ai_disclosure` | AI 生成声明 |
| `model_license` | 模型许可证或服务条款记录 |
| `reference_sources` | 参考来源摘要，不保存侵权复制内容 |
| `export_allowed` | 是否允许导出交付 |
| `platform_profile_id` | 可选外部平台连接配置 |
| `manual_approval_required` | 是否需要人工批准 |

## 12. 第一阶段实际目标

第一阶段不接真实大模型也可以先完成产品骨架和流程闭环。

必须完成：

- 网站应用骨架。
- 每日任务页面。
- 作品库页面。
- 评分中心页面。
- 交付与授权中心页面。
- Mock 数据贴近真实生产流程。
- 文档化的 LOOP 状态流转。
- 文档化的质量评分体系。

暂不完成：

- 真实音乐生成模型训练。
- 完整平台自动上传。
- 社区和播放平台。
- 收益统计平台。
- 复杂权限系统。

## 13. 成功标准

本阶段成功不是“生成一首歌”，而是证明系统结构能支撑持续制作。

验收时应能回答：

- 今天系统要做哪 10-20 条音乐制作任务？
- 每条任务处于哪个制作阶段？
- 每首作品有哪些版本？
- 为什么某个版本通过或失败？
- 失败后如何自动返工？
- 哪些作品已经进入资产库？
- 哪些作品可以生成交付包？
- 授权配置是否完整？
- 哪些内容需要人工复核？
